## Epic 1 — Надёжность и эксплуатация

### [P0] Единый формат operational-логов
**Labels:** `epic`, `infra`, `p0`

**Описание**
Сделать единый формат логов для операций: генерация вопроса, веб-поиск, старт/стоп игры, таймаут вопроса, отправка media-раунда.

**AC**
- [ ] В логах есть `chat_id`, `operation`, `duration_ms`, `result`, `error_type`.
- [ ] Ошибки внешних API пишутся с контекстом операции.
- [ ] Формат одинаковый во всех ключевых местах.

**Estimate:** 6h

---

### [P0] Retry + backoff для внешних API
**Labels:** `backend`, `infra`, `p0`

**Описание**
Добавить retry для OpenAI/Yandex: повторять только сетевые/429/5xx ошибки, не повторять валидационные ошибки.

**AC**
- [ ] Максимум 3 попытки.
- [ ] Экспоненциальная пауза между попытками.
- [ ] После исчерпания retry пользователю отправляется мягкое сообщение.

**Estimate:** 6h
**Depends on:** Единый формат operational-логов
